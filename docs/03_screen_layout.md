# 주식 자동매매 프로그램 화면 구성 정리

## 0. 기본 화면 구조

화면은 탭 구조로 구성한다.

기본 탭:
1. 전일 상친 목록
2. 내 보유 목록
3. 주문 예상 목록
4. 체결/주문 목록
5. 주문 목록 상세 탭

공통 규칙:
- 종목을 클릭하면 주문 목록 상세 탭을 연다.
- 주문 목록 상세 탭은 종목코드 기준으로 중복으로 열지 않는다.
- 이미 열린 종목이면 새 탭을 만들지 않고 기존 탭으로 이동한다.
- REST API는 주문, 잔고, 체결확인용으로 사용한다.
- WebSocket은 실시간 시세, 호가, 예상체결가 감시용으로 사용한다.

## 1. 전일 상친 목록 탭

목적:
전일 상한가 종목을 수집하고, 당일 매수 후보를 선별한다.

표시 대상:
- 코스피 기준 전일 상한가 종목
- 필요 시 코스닥도 옵션으로 추가 가능
- 예: 최대 100건

표시 정보:
- 종목코드
- 종목명
- 전일 종가
- 전일 등락률
- 전일 거래량
- 전일 거래대금
- 시가총액
- PER
- PBR
- EPS
- ROE
- 유통주식수
- 거래량 증가율
- 테마/섹터
- 투자주의/투자경고 여부
- 거래정지 여부
- 단기과열 여부
- 조건 점수

화면 기능:
- 전일 상친 목록 조회
- 종목별 기본 정보 수집
- 조건 점수 계산
- 조건 기준 상위 10개 종목을 리스트 상단에 노출
- 종목 클릭 시 주문 목록 상세 탭 열기
- 주문 예상 목록 담기 버튼
- 새로고침 버튼
- 조건 재계산 버튼


------------------------------------------------------------
## 2. 내 보유 목록 탭


목적:
내 프로그램이 매수한 보유 종목만 관리한다.

표시 대상:
- 당일 프로그램으로 매수한 종목
- 전날부터 넘어온 프로그램 보유 종목
- 미매도 잔량이 있는 종목

제외 대상:
- 사용자가 직접 손으로 매수한 종목
- 프로그램 주문 이력이 없는 종목

표시 정보:
- 종목코드
- 종목명
- 보유수량
- 매수 평균가
- 현재가
- 평가손익
- 수익률
- 매수일
- 주문 전략명
- 자동매도 감시 여부
- 현재 매도 주문 상태
- 미체결 수량

화면 기능:
- 종목 클릭 시 주문 목록 상세 탭 열기
- 주문 예상 목록 담기 버튼
- 자동매도 시작 버튼
- 자동매도 중지 버튼
- 전량 매도 버튼
- 보유 종목 동기화 버튼


------------------------------------------------------------
## 3. 주문 예상 목록 탭

목적:
당일 실제 주문할 후보 종목을 관리한다.

표시 대상:
- 전일 상친 목록에서 담은 종목
- 내 보유 목록에서 담은 종목
- 사용자가 직접 추가한 종목

표시 정보:
- 종목코드
- 종목명
- 전일 종가
- 현재 예상체결가
- 예상체결량
- 전일 대비 예상 등락률
- 매수 예정 수량
- 매수 예정 금액
- 최대 허용 매수가
- 매수 조건 충족 여부
- 자동매수 여부
- 상태

상태 예시:
- 대기
- 감시중
- 조건충족
- 주문전송
- 매수체결
- 매수실패
- 제외

화면 기능:
- 종목 클릭 시 주문 목록 상세 탭 열기
- 후보 삭제
- 자동매수 ON/OFF
- 최대 매수가 설정
- 매수 수량 설정
- 감시 시작 버튼
- 감시 중지 버튼
- 전체 감시 시작 버튼
- 전체 감시 중지 버튼

## 4. 체결/주문 목록 탭

목적:
당일 발생한 매수/매도 주문과 체결 내역을 관리한다.

표시 대상:
- 당일 프로그램이 넣은 주문
- 당일 프로그램 주문의 체결 내역
- 매수 주문
- 매도 주문
- 취소 주문
- 정정 주문

표시 정보:
- 주문시간
- 종목코드
- 종목명
- 주문구분
- 매수/매도 구분
- 주문가격
- 주문수량
- 체결수량
- 미체결수량
- 평균체결가
- 주문상태
- 주문번호
- 원주문번호
- 전략명

주문상태 예시:
- 주문대기
- 주문전송
- 접수완료
- 부분체결
- 전량체결
- 미체결
- 취소요청
- 취소완료
- 정정요청
- 정정완료
- 주문실패

화면 기능:
- 종목 클릭 시 주문 목록 상세 탭 열기
- 미체결 주문 취소
- 미체결 주문 정정
- 체결 내역 새로고침
- 주문 로그 확인

## 6. 추가하면 좋은 화면

## 6-1. 자동매매 로그 탭

목적:
자동매매 판단과 주문 실행 기록을 확인한다.

표시 정보:
- 시간
- 종목코드
- 종목명
- 이벤트 종류
- 판단 조건
- 예상체결가
- 현재가
- 주문가격
- 주문수량
- 결과
- 에러 메시지

이벤트 예시:
- 후보등록
- 감시시작
- 조건충족
- 조건불충족
- 주문전송
- 주문성공
- 주문실패
- 체결확인
- 매도감시시작
- 손절조건도달
- 전략종료


## 6-2. 설정 탭

목적:
자동매매 조건을 화면에서 변경할 수 있게 한다.

설정 항목:
- 1회 최대 매수 금액
- 종목당 최대 매수 금액
- 하루 최대 매수 금액
- 하루 최대 손실 금액
- 목표 수익률
- 손절률
- 자동매수 시작 시간
- 자동매수 종료 시간
- 자동매도 강제 종료 시간
- 코스피만 볼지 여부
- 코스닥 포함 여부
- 투자주의 종목 제외 여부
- 거래정지 종목 제외 여부
- 단기과열 종목 제외 여부


## 6-3. 위험 제어 탭

목적:
자동매매 중 긴급 상황을 제어한다.

화면 기능:
- 전체 자동매매 중지 버튼
- 전체 미체결 주문 취소 버튼
- 전체 보유 종목 전량 매도 버튼
- 오늘 손익 확인
- 오늘 손실 한도 확인
- API 연결 상태 확인
- WebSocket 연결 상태 확인

자동 중지 조건:
- 오늘 손실 한도 도달
- API 오류 반복 발생
- WebSocket 연결 끊김
- 잔고 조회 실패
- 주문 응답 실패


## 6-4. 종목 제외 목록 탭

목적:
자동매매 대상에서 제외할 종목을 관리한다.

표시 정보:
- 종목코드
- 종목명
- 제외 사유
- 등록일

제외 사유 예시:
- 투자주의
- 투자경고
- 거래정지
- 단기과열
- 관리종목
- 거래대금 부족
- 유동성 부족
- 사용자 직접 제외

화면 기능:
- 제외 종목 추가
- 제외 종목 삭제
- 제외 사유 입력
