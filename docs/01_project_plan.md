# 주식 프로그램 만들기 기획서

## 1. 프로젝트 개요

### 프로젝트명

주식 뉴스 기반 선제 대응 프로그램

### 목적

한국 경제에 영향을 줄 수 있는 주요 경제 뉴스를 주기적으로 수집하고, 관련 국내 주식 종목을 분석하여 장 시작 전 또는 장중에 매수·매도 판단을 빠르게 할 수 있도록 돕는 프로그램을 만든다.

특히 IT, 방산 등 뉴스 민감도가 높은 업종을 중심으로 경제 뉴스와 종목 간의 연관성을 추적하고, 사용자가 선제적으로 대응할 수 있는 투자 판단 보조 시스템을 구축하는 것이 목표다.

### 핵심 목표

- 주요 경제 뉴스 자동 수집
- 뉴스 요약 및 중요도 판단
- 관련 국내 주식 종목 추출
- 종목별 영향 분석
- Google Sheets에 일별 기록
- 메신저 알림 전송
- 향후 자동 매수·매도 기능 확장

---

## 2. 문제 정의

현재 주식 투자 판단은 뉴스를 직접 찾아보고, 관련 종목을 직접 연결하고, 매수·매도 여부를 사람이 일일이 판단해야 한다.

하지만 경제 뉴스는 빠르게 퍼지고 시장은 즉시 반응하기 때문에, 장 시작 전이나 장중에 빠르게 정리된 정보를 확인할 수 있다면 대응 속도를 높일 수 있다.

이 프로그램은 “뉴스를 먼저 정리하고, 관련 종목을 빠르게 찾고, 내가 살지 말지 판단할 수 있게 돕는 것”에 초점을 둔다.

---

## 3. MVP 범위

초기 버전에서는 자동매매까지 바로 구현하지 않고, 뉴스 수집·종목 분석·시트 정리·알림까지를 MVP로 잡는다.

### MVP 포함 기능

1. 경제 뉴스 수집
2. 뉴스 요약
3. 관련 종목 추출
4. 종목별 간단 분석
5. Google Sheets에 일별 저장
6. 매수 관심 종목 체크
7. 예상 매수 금액 및 수량 입력
8. Telegram, Discord, 이메일 등 알림 발송

### MVP 제외 기능

1. 실제 자동 매수·매도
2. 실시간 초단타 매매
3. 복잡한 차트 자동 분석
4. 완전 자동 투자 판단
5. 수익률 보장형 추천

---

## 4. 전체 시스템 흐름

경제 뉴스 수집
→ 뉴스 요약 및 중요도 판단
→ 관련 업종 / 종목 추출
→ 종목별 영향 분석
→ Google Sheets 저장
→ 사용자 확인 및 체크
→ 알림 전송
→ 향후 자동매매 기능 연동

## 5. 에이전트 구성

### 에이전트 1. 경제 뉴스 수집 에이전트

#### 역할

한국 경제와 국내 주식시장에 영향을 줄 수 있는 주요 뉴스를 주기적으로 수집한다.

#### 주요 기능

- 경제 뉴스 수집
- 산업별 뉴스 분류
- 중요 뉴스 선별
- 뉴스 제목, 출처, 링크, 발행 시간 저장
- Google Docs 또는 Google Sheets에 일별 기록

#### 추천 수집 주기

- 06:30: 장 시작 전 사전 점검
- 12:00: 오전장 반응 확인
- 15:30: 장 마감 직후 정리
- 21:00: 해외 이슈 및 다음 날 준비

---

### 에이전트 2. 종목 분석 에이전트

#### 역할

수집된 뉴스와 관련 있는 국내 주식 종목을 찾고, 해당 종목에 미칠 영향을 분석한다.

#### 주요 기능

- 뉴스와 관련된 업종 추론
- 관련 종목 후보 추출
- 긍정 / 부정 / 중립 영향 분류
- 단기 대응 여부 판단
- 매수 후보, 매도 후보, 관망 후보 분류
- Google Sheets에 일별 목록 저장

#### Google Sheets 예시 컬럼

날짜 | 뉴스 제목 | 관련 업종 | 관련 종목 | 영향 | 중요도 | 대응 의견 | 매수 체크 | 예상 금액 | 예상 수량 | 비고

---

### 에이전트 3. 알림 에이전트

#### 역할

Google Sheets에 정리된 중요 종목과 분석 내용을 사용자가 바로 확인할 수 있도록 메신저로 전송한다.

#### 주요 기능

- 중요도 높은 뉴스만 필터링
- 매수 체크된 종목 요약
- 장 시작 전 요약 알림
- 장중 긴급 뉴스 알림
- Telegram, Discord, 이메일, 카카오톡 중 선택하여 전송

#### 알림 예시

[오늘의 주식 뉴스 요약]

1. 방산 수출 확대 뉴스

- 관련 종목: 한화에어로스페이스, LIG넥스원
- 영향: 긍정
- 대응: 관심 / 장 초반 변동성 확인

2. 반도체 규제 완화 뉴스

- 관련 종목: 삼성전자, SK하이닉스
- 영향: 긍정
- 대응: 관망 후 진입 검토

---

### 에이전트 4. 자동매매 에이전트

#### 역할

향후 에이전트 2에서 정리한 분석 데이터를 바탕으로 주식 매수·매도를 수행한다.

#### 주의사항

자동매매는 실제 돈이 움직이는 기능이므로 MVP에서는 제외하고, 충분한 테스트 이후 별도 단계로 개발한다.

#### 향후 기능

- 증권사 API 연동
- 매수 조건 설정
- 매도 조건 설정
- 손절 / 익절 기준 설정
- 1일 최대 매수 금액 제한
- 종목별 최대 투자 비중 제한
- 실제 주문 전 사용자 승인 단계

---

## 6. 추천 기술 구조

### 1단계: 수동 + 반자동 구조

#### 목적

처음부터 자동매매까지 가지 않고, 뉴스 수집과 정리 자동화부터 만든다.

#### 구성

- 뉴스 수집: Python 또는 Node.js
- AI 요약: OpenAI API 또는 Claude API
- 저장: Google Sheets
- 문서화: Google Docs
- 알림: Telegram, Discord, 이메일, 카카오톡 중 선택

---

### 2단계: 웹 화면 추가

#### 목적

Google Sheets만으로 관리가 불편해지면 별도 웹 화면을 만든다.

#### 구성

- Frontend: React / Next.js
- Backend: Spring Boot 또는 Node.js
- DB: PostgreSQL 또는 MySQL
- 배치 실행: cron, GitHub Actions, 서버 스케줄러

---

### 3단계: 자동매매 연동

#### 목적

분석 결과를 바탕으로 실제 주문까지 연결한다.

#### 구성

- 증권사 Open API
- 주문 전 검증 로직
- 매수·매도 조건 관리
- 주문 로그 저장
- 실패 시 알림
- 일일 손실 제한

## 10. 리스크 및 주의사항

### 투자 판단 리스크

AI 분석 결과는 투자 참고용으로만 사용해야 하며, 매수·매도 결정은 최종적으로 사용자가 직접 해야 한다.

### 자동매매 리스크

자동매매는 오류 발생 시 실제 손실로 이어질 수 있으므로, 반드시 모의투자와 제한 조건을 먼저 적용해야 한다.

### 데이터 품질 리스크

뉴스 수집 출처가 부정확하거나 중복 뉴스가 많으면 분석 결과의 품질이 떨어질 수 있다.

### API 비용 리스크

AI API를 자주 호출하면 비용이 증가할 수 있으므로, 뉴스 중복 제거와 중요도 필터링이 필요하다.

## 11. 최종 방향

이 프로젝트는 처음부터 완전 자동매매 프로그램으로 만들기보다, 먼저 “뉴스 기반 투자 판단 보조 도구”로 시작하는 것이 좋다.

1차 목표는 장 시작 전 확인할 수 있는 뉴스·종목 요약표를 자동으로 만드는 것이다.

이후 충분한 데이터가 쌓이면, 종목별 반응 패턴을 분석하고, 모의투자 기능을 붙인 뒤, 마지막 단계에서 자동매매를 검토한다.
